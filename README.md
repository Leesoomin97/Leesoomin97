**Stack:** Python, PyTorch, HuggingFace Transformers, mBART, Pandas, Weights & Biases  

#### Highlights
- 대화 요약 데이터의 **길이·주제 불균형 문제를 EDA로 분석**하고, 단순 Fine-tuning이 아닌 **일반화 중심 학습 전략**으로 문제를 재정의
- Dialogue 평균 406자 vs Summary 평균 86자의 **극단적인 Length Mismatch**를 고려해 **1024 토큰 Long-context 처리 + 압축 성능 최적화**에 집중
- Train–Dev 간 **Topic 중복률이 31%**에 불과한 구조적 문제를 발견하고 Topic 정보를 입력에서 제외하여 **주제 암기 대신 요약 구조 학습**을 유도
- KoBART 대비 영어·한글 혼합 텍스트에서 **Tokenizer 분절 효율이 우수한 mBART-50** 채택
- **Curriculum Learning 기반 3-Stage 학습 전략** 설계  
  - Phase 1: SAMSum(영어 대화 요약) 데이터로 요약 논리 학습  
  - Phase 2: AI Hub 한국어 데이터로 문체 및 언어 적응  
  - Phase 3: 대회 데이터 + Solar API 기반 증강 데이터로 최종 파인튜닝
- R3F 실험, K-Fold 학습, 화자 태그 보정 및 띄어쓰기 교정 등 후처리를 결합한 **Recursive Ensemble 전략**으로 성능 극대화
- 🏆 **Final Leaderboard:** 🥈 2위 (ROUGE-1/2/L 평균 **47.79**)

### 🔬 [Scientific Knowledge Question Answering – Retrieval System (RAG)](https://github.com/AIBootcamp16/scientific-knowledge-question-answering-ir-4)
**Type:** Information Retrieval, RAG – Competition Project (Team)  
**Goal:** 과학 상식 질의에 대해 가장 관련성 높은 문서를 정확하게 검색하는 IR 파이프라인 설계  
**Stack:** Python, Elasticsearch, PyTorch, HuggingFace Transformers, Sentence-Transformers, GPT-4o-mini  

#### Highlights
- 과학 상식 질의 응답 문제를 **End-to-End 생성 문제가 아닌 Retrieval 중심 문제로 재정의**하고 최종 생성 성능보다 **Retrieval 정확도(MAP@3)** 최적화에 집중
- 질의 데이터에 **비지식성 Small Talk가 다수 포함**된 구조적 문제를 발견하고 검색 생략을 위한 **Smalltalk Guard 로직** 설계
- 대화형 질의를 단순 검색어가 아닌 **Standalone Query 생성 프롬프트**로 변환하여 대화 맥락을 반영
- Sparse 검색의 한계를 보완하기 위해 **BM25(Nori) + Dense Embedding(E5, KURE-v1)** 기반 **Hybrid Search** 적용
- Original / Expanded / Conceptual Query를 결합한 **Multi-query Retrieval** 전략 설계
- 초기 Recall 확보를 위해 **후보군을 대폭 확장(num_candidates ≈ 1,200)**하고 **2-Stage Reranker(BGE-reranker-v2-m3 + Korean Reranker 앙상블)**로 Precision 회복
- RRF(Reciprocal Rank Fusion)를 활용한 검색 결과 통합으로 도메인 편중 데이터에서도 안정적인 성능 확보
- 🏆 **Final Performance:** MAP@3 **0.8970**  
  (Baseline 대비 **약 +21% 성능 향상**)

### 🏛️ [Government Policy & Subsidy AI Agent (LangGraph / RAG)](https://github.com/AIBootcamp16/langchain-llm-langchain-3)
**Type:** AI Agent, RAG, Full-Stack Web Service – Team Project  
**Goal:** 정부 정책·지원금 정보를 쉽게 탐색하고, **근거 기반 설명 + 자격 가능성 판단**까지 제공하는 AI 에이전트 서비스 구축  
**Stack:** Python, FastAPI, LangGraph, OpenAI GPT-4o-mini, MySQL, Qdrant, BGE-M3, BM25, Next.js, Docker, LangSmith  

#### Highlights
- 정부 정책 탐색 문제를 단순 Q&A가 아닌 **Search → Reasoning → Eligibility Check**로 분리된 **Agent 기반 문제**로 재정의
- **Hybrid Search 파이프라인** 설계  
  - Dense(BGE-M3) + Sparse(BM25) + RRF 기반 검색  
  - 지역·카테고리·대상 그룹 필터링 및 **웹 검색(Tavily) 자동 보강**
- LangGraph 기반 **조건부 라우팅 Q&A Agent** 구현  
  - 문서만으로 답변 가능 여부 판단  
  - 문서 부족 시 웹 검색 → 하이브리드 답변 자동 전환
- 모든 답변에 **정책 문서 / 웹 출처를 명시하는 근거 기반 응답(Citation)** 구조 적용
- **Eligibility Check Agent** 설계  
  - 정책 자격 조건을 구조화하여 파싱  
  - 대화형 체크리스트 질문 생성  
  - 사용자 응답을 기반으로 **ELIGIBLE / NOT_ELIGIBLE / UNKNOWN** 자동 판정
- **SSE(Server-Sent Events)** 기반 실시간 스트리밍 응답으로 사용자 경험 개선
- 정책 문서·대화 히스토리 **인메모리 캐싱**으로 검색 비용 및 응답 지연 최소화
- LangSmith 기반 **Observability & Evaluation** 구축  
  - QA / Search / Eligibility 전 워크플로우 트레이싱  
  - Groundedness, Citation Rate, Response Time 등 정량 평가 수행
- Docker 기반 **로컬–서버 동일 환경** 구성 및 Backend(Cloudtype) / Frontend(Vercel) 분리 배포
